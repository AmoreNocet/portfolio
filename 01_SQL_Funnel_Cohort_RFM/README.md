# SQL-проект: Анализ воронки, когортный анализ и RFM-сегментация

## Бизнес-кейс

**Контекст:** SaaS-продукт с freemium-моделью. Команда фиксирует высокий показатель регистраций, но конверсия в платящих пользователей остаётся низкой (~4%). Product Manager ставит задачу: найти «узкие места» в воронке активации и определить, какие сегменты пользователей наиболее ценны.

**Вопросы для анализа:**
1. Где именно теряются пользователи в воронке регистрация → активация → первая оплата → повторная оплата?
2. Как выглядит retention по когортам: улучшился ли он за последние 6 месяцев?
3. Кто наши лучшие клиенты по RFM, и как сфокусировать ретенционные усилия?

**Ожидаемые результаты:**
- Приоритизация точек роста для продуктовой команды
- Выделение высокоценных сегментов для CRM-кампаний
- Базовая метрика retention для трекинга влияния продуктовых изменений

---

## Стек

- **SQL**: PostgreSQL (синтаксис совместим с ClickHouse, с минимальными правками)
- **Техники**: Window Functions, CTEs, Date Math, CASE WHEN, FILTER

## Схема данных

```sql
-- Пользователи
users (user_id, registered_at, plan_type, country)

-- События (активация, использование фич)
events (event_id, user_id, event_type, created_at)
-- event_type: 'registration', 'onboarding_complete', 'feature_used', 'invite_sent'

-- Платежи
payments (payment_id, user_id, amount, paid_at, status)
-- status: 'success', 'failed', 'refunded'
```

## Интерпретации

- **Воронка**: 100% регистрируются → 68% завершают онбординг → 31% используют ключевую фичу → 12% начинают пробный период → 4% конвертируются в оплату.
  Главный дроп: -37% между онбордингом и использованием ключевой фичи. **Гипотеза:** UX проблема в шаге активации.

- **Retention**: когорты последних 3 месяцев показывают M1 retention +5 п.п. относительно 6-месячной давности — эффект от недавнего продуктового обновления.

- **RFM**: сегмент "Champions" (~8% базы) даёт 34% выручки. Сегмент "At Risk" (~15% базы) имеет LTV сопоставимый с Champions, но не совершал покупок >60 дней — приоритет для win-back кампании.

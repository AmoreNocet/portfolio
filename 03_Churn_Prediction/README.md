# Python-проект: Предсказание оттока клиентов (Churn Prediction)

## Бизнес-кейс

**Контекст:** SaaS-компания теряет ~18% платящих пользователей ежемесячно. CAC (стоимость привлечения) значительно выше стоимости удержания. Продуктовая команда хочет внедрить проактивную ретенционную механику: за 2 недели до ожидаемого оттока пользователь получает персональное предложение.

**Задача:** Построить ML-модель, которая предскажет, кто из текущих активных пользователей уйдёт в течение следующих 30 дней.

**Метрики успеха модели:**
- ROC-AUC ≥ 0.75 (достаточная разделяемость)
- Precision в топ-20% пользователей ≥ 60% (эффективность кампании)

**Бизнес-метрики:**
- ROI ретенционной кампании на основе предсказаний
- Ожидаемое снижение churn rate на 3–5 п.п.

## Что делает ноутбук

1. **Генерация данных** — синтетический датасет с реалистичными поведенческими фичами
2. **EDA** — анализ распределений, корреляций, class imbalance
3. **Feature Engineering** — агрегированные фичи активности, тренды, RFM-признаки
4. **Моделирование** — Logistic Regression baseline + Random Forest + порог классификации
5. **Оценка** — ROC-AUC, Precision@K, confusion matrix, lift curve
6. **Интерпретация** — feature importance, бизнес-интерпретация топ-признаков
7. **ROI-расчёт** — перевод модельных метрик в бизнес-ценность

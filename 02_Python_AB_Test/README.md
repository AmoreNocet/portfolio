# Python-проект: Анализ A/B-теста — новый онбординг vs контроль

## Бизнес-кейс

**Контекст:** Продуктовая команда редизайнила онбординг-флоу: упростила шаги, добавила прогресс-бар и интерактивный туториал. Гипотеза: новый онбординг повысит долю пользователей, завершивших активацию в течение 7 дней после регистрации.

**Метрика успеха (Primary metric):** 7-day Activation Rate — доля пользователей, выполнивших ключевое целевое действие (feature_used) в течение 7 дней.

**Guardrail metrics:** время на онбординг, процент пользователей, оставивших онбординг незавершённым (dropout rate).

**Период эксперимента:** 28 дней, random split 50/50 по user_id.

## Что делает ноутбук

1. **Генерация данных** — реалистичный синтетический датасет с известным эффектом
2. **EDA** — базовая проверка данных, распределения, размеры групп
3. **Расчёт MDE и мощности** — постфактум и как проверка дизайна теста
4. **Статистический тест** — z-test для пропорций + bootstrap CI
5. **Проверка SRM** (Sample Ratio Mismatch) — на корректность разбивки
6. **Визуализация** — воронка, конверсии по дням, bootstrap-распределение
7. **Бизнес-интерпретация** — перевод stат-значимости в деньги и решение по масштабированию
